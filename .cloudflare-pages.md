# Configuration Cloudflare Pages - Styliste.com

## Compression automatique

Cloudflare Pages applique automatiquement :
- **Brotli compression** (prioritaire si supporté par le navigateur)
- **Gzip compression** (fallback pour les anciens navigateurs)

### Gains estimés avec Brotli
- JavaScript : ~15-20% plus petit que gzip
- Bundle estimé : **~45 KB** (vs 53 KB gzipped)
- CSS : ~20-25% plus petit que gzip

### Configuration des headers

Cloudflare Pages ajoute automatiquement :
```
Content-Encoding: br
Vary: Accept-Encoding
Cache-Control: public, max-age=31536000, immutable  (pour assets hashés)
```

## Autres optimisations Cloudflare

### CDN Global
- POPs en Afrique : Lagos (Nigeria), Johannesburg (Afrique du Sud)
- Temps de réponse estimé : 50-150ms depuis le Bénin

### Cache
- **Assets statiques** (JS, CSS, images) : Cache permanent
- **Pages HTML** : Cache avec revalidation
- **API routes** : Pas de cache (dynamic)

### Auto-minify
Cloudflare Pages peut également minifier :
- HTML
- CSS
- JavaScript

**Note :** Déjà fait par Vite, donc redondant mais sans impact négatif.

## Configuration recommandée

Lors du déploiement sur Cloudflare Pages :

**Build settings :**
- Build command : `npm run build`
- Build output directory : `.svelte-kit/output/client`
- Node version : 20.x

**Environment variables :**
```
PUBLIC_SUPABASE_URL=https://your-project.supabase.co
PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key  (via Secrets)
DATABASE_URL=postgresql://...  (via Secrets)
```
